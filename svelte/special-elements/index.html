    MEMO IN BRAIN SVELTE SPECIAL-ELEMENTS

    Svelte Special elements v3.49.0

    https://svelte.dev/docs

    self

    https://svelte.dev/docs#template-syntax-svelte-self

    <svelte:self> 는 자기 자신을 재귀적으로 참조할 때 사용한다.
    트리뷰와 같이 요소를들 반복해서 참조해야 할 때 사용한다.

    component

    https://svelte.dev/docs#template-syntax-svelte-component

    <svelte:component> 로 구성요소 생성자를 사용하여 구성요소를 동적으로 렌더링한다.
    속성이 변경되면 구성요소가 삭제되고 재생성된다.

    element

    https://svelte.dev/docs#template-syntax-svelte-element

    <svelte:element> 는 동적으로 지정된 유형의 요소로 렌더링한다.

    window

    https://svelte.dev/docs#template-syntax-svelte-window

    <svelte:window> 는 window 객체에 접근할 때 사용한다.

    head

    https://svelte.dev/docs#template-syntax-svelte-head

    <svelte:head> 는 head 요소에 접근할 때 사용한다.

    body

    https://svelte.dev/docs#template-syntax-svelte-body

    <svelte:body> 는 body 요소에 접근할 때 사용한다.

    options

    https://svelte.dev/docs#template-syntax-svelte-options

    <svelte:options> 는 컴파일에 대한 사용자 지정 옵션이다.
    immutable true 일 경우 컴파일러가 동일성을 검사하여 확실히 값이 변경되었을 때만 동작하게 함
    accessors 컴포넌트 Props 에 getter, setters 를 추가함
    namespace 일반적으로 사용될 네임스페이스
    tag 컴파일 시 커스텀 요소의 이름 지정

    fragment

    https://svelte.dev/docs#template-syntax-svelte-fragment

    <svelte:fragment> 는 slot 를 사용할 때 불필요한 요소를 생성하지 않도록 한다.
