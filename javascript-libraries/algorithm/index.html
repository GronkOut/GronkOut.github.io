    Javascript - Algorithm

    Fibonacci

    n-2 번째와 n-1 번째 수를 합하여 n 번째 수로 나타내는 수열을 피보나치 수열이라 한다.
    재귀 형태로 구현 시 불필요한 중복 호출이 발생하여 런타임이 길어진다.
    배열에 결과를 누적하여 처리하는 방법으로 구현하면 성능이 좋다.

    Debounce

    특정 시간이 지난 후 하나의 이벤트만 발생시킨다.
    추천 검색어 API 호출에 적절하다.

    Throttle

    일정한 시간마다 이벤트를 발생시킨다.
    무한 스크롤링 UI 에 적절하다.

    Divisor

    약수를 구한다.
    자연수가 제곱수이면 약수는 홀수개이고, 제곱수가 아니면 약수는 짝수개이다.

    Prime

    소수인지 판별한다.

    GCD

    최대 공약수를 구한다.
    2 부터 min(a, b) 까지 모든 정수로 나누어 본다.

    LCM

    최소 공배수를 구한다.
    두 수를 result 로 나누었을 때 나머지 값이 0 인지 확인한다.

    Combinations

    [조합] 경우의 수에서 순서는 무시한다.

    Permutations

    [순열] 경우의 수에서 순서도 고려한다.

    Array traversal

    Breadth First Search (BFS)

    Depth First Search (DFS)

    Object traversal

    Breadth First Search (BFS)

    Depth First Search (DFS)

    Graph traversal

    공항 경유 포인트를 그래프로 생성한다.

    Breadth First Search (BFS)

    너비 우선 탐색을 한다.
    최단 경로를 찾기 위해 큐 데이터 구조를 사용한다.
    FIFO (First In First Out) 선입 선출 개념에 따라 작동한다.
    메모리가 많이 필요하고 느리다.

    Depth First Search (DFS)

    깊이 우선 탐색을 한다.
    스택 구조를 사용한다.
    LIFO (Last In First Out) 후입 선출 개념에 따라 작동한다.
    메모리가 적게 필요하고 빠르다.
